# 📝 Day 2 (03/31) — JOIN


## 🔥 오늘 배운 핵심 한 줄 요약

> **조인 = 두 개 이상의 테이블을 연결해서 하나의 결과로 합치는 것**

---

## 1. 왜 조인이 필요한가?

- 데이터를 여러 테이블에 나눠서 저장함 (정규화)
- 하나의 결과를 보려면 테이블을 합쳐야 함
- 예: 주문 테이블 + 고객 테이블 → "누가 뭘 주문했는지"

---

## 2. 내부 조인 (INNER JOIN)

> **양쪽 테이블 모두에 매칭되는 행만** 가져온다

```sql
SELECT *
FROM 주문 o
INNER JOIN 고객 c ON o.고객ID = c.고객ID;
```

- 매칭 안 되는 행은 결과에서 **제외**됨
- 가장 기본적이고 많이 쓰이는 조인

---

## 3. 외부 조인 (OUTER JOIN)

### LEFT JOIN 

> **왼쪽 테이블은 전부** 가져오고, 오른쪽에 매칭 없으면 NULL

```sql
SELECT *
FROM 고객 c
LEFT JOIN 주문 o ON c.고객ID = o.고객ID;
```

- 주문 안 한 고객도 결과에 포함됨 (주문 정보는 NULL)
- **"없는 것도 포함해서 보고 싶을 때"** 사용

### RIGHT JOIN

> **오른쪽 테이블은 전부** 가져오고, 왼쪽에 매칭 없으면 NULL

```sql
SELECT *
FROM 주문 o
RIGHT JOIN 고객 c ON o.고객ID = c.고객ID;
```

- LEFT JOIN에서 테이블 순서를 바꾼 것과 동일
- 실무에서는 LEFT JOIN을 주로 씀

### FULL OUTER JOIN

> **양쪽 모두** 전부 가져오고, 매칭 없으면 NULL

- MySQL에서는 직접 지원 안 함 → UNION으로 구현

---

## 4. 셀프 조인 (SELF JOIN)

> **같은 테이블을 자기 자신과** 조인

```sql
SELECT e.이름 AS 직원, m.이름 AS 매니저
FROM 직원 e
JOIN 직원 m ON e.매니저ID = m.직원ID;
```

- 같은 테이블 안에서 관계가 있을 때 사용
- 예: 직원-매니저 관계, 카테고리-상위카테고리

---

## 5. CROSS JOIN (교차 조인)

> **모든 경우의 수** 조합 (A × B)

```sql
SELECT *
FROM 색상
CROSS JOIN 사이즈;
```

- 색상 3개 × 사이즈 4개 = 12행
- ON 조건 없음

---

## 6. 조인 한눈에 비교

| 조인 종류 | 특징 | 매칭 없을 때 |
|-----------|------|-------------|
| INNER JOIN | 양쪽 다 있는 것만 | 제외 |
| LEFT JOIN | 왼쪽 전부 + 오른쪽 매칭 | 오른쪽 NULL |
| RIGHT JOIN | 오른쪽 전부 + 왼쪽 매칭 | 왼쪽 NULL |
| SELF JOIN | 자기 자신과 조인 | 조인 타입에 따라 다름 |
| CROSS JOIN | 모든 조합 | 조건 자체가 없음 |

---

## 7. 조인의 특징 (핵심!)

- **별칭(Alias) 사용**: 테이블명이 길면 `AS` 또는 약어 사용
```sql
FROM 고객 c JOIN 주문 o ON c.id = o.customer_id
```

- **여러 테이블 조인 가능**: 2개뿐 아니라 3개, 4개도 가능
```sql
FROM 주문 o
JOIN 고객 c ON o.고객ID = c.고객ID
JOIN 상품 p ON o.상품ID = p.상품ID;
```

- **ON 조건이 핵심**: 어떤 컬럼으로 연결하는지가 조인의 전부

---

## 8. 실무에서 자주 쓰는 패턴

### LEFT JOIN으로 "없는 것" 찾기
```sql
-- 주문을 한 번도 안 한 고객 찾기
SELECT c.*
FROM 고객 c
LEFT JOIN 주문 o ON c.고객ID = o.고객ID
WHERE o.주문ID IS NULL;
```

### 여러 조인 + 집계
```sql
-- 부서별 직원 수
SELECT d.부서명, COUNT(e.직원ID) AS 직원수
FROM 부서 d
LEFT JOIN 직원 e ON d.부서ID = e.부서ID
GROUP BY d.부서명;
```

---

## 📌 오늘의 핵심 포인트

1. **INNER JOIN** = 교집합 (둘 다 있는 것만)
2. **LEFT JOIN** = 왼쪽 전부 + 매칭 (실무에서 제일 많이 씀!)
3. **ON 조건** = 테이블을 어떻게 연결할지 결정하는 핵심
4. **LEFT JOIN + IS NULL** = "없는 것 찾기" 패턴 (면접 단골!)
5. **조인은 여러 개** 연결 가능 (3개, 4개도 OK)
