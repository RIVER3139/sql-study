# 📝 Day 6 (04/04) — 인덱스 기초

## 🔥 오늘 배운 핵심 한 줄 요약

> **인덱스 = 데이터를 빠르게 찾기 위한 목차. 책의 색인과 같은 원리**

---

## 1. 인덱스가 필요한 이유

- 테이블에 데이터가 수만~수억 건일 때, 매번 전체를 훑으면 느림
- 이걸 **풀 테이블 스캔(Full Table Scan)**이라고 함
- 인덱스가 있으면 **원하는 데이터를 바로 찾아감** → 성능 대폭 향상

---

## 2. 인덱스의 원리 — 트리 자료 구조

### B-Tree (Balanced Tree)

- 인덱스는 **B-Tree 구조**로 되어 있음
- 데이터가 정렬된 상태로 저장됨
- 검색 시 **위에서 아래로 내려가면서** 범위를 좁혀나감
- 시간 복잡도: O(log N) → 데이터가 100만 건이어도 약 20번만에 찾음

```
         [50]
        /    \
    [30]      [70]
    /  \      /  \
 [10] [40] [60] [80]
```

> 💡 핵심: 인덱스 = **정렬된 데이터 + 트리 구조 = 빠른 검색**

---

## 3. 인덱스 생성, 조회, 삭제

### 인덱스 생성

```sql
-- 단일 컬럼 인덱스
CREATE INDEX idx_이름 ON 직원(이름);

-- 유니크 인덱스 (중복 불가)
CREATE UNIQUE INDEX idx_이메일 ON 회원(이메일);
```

### 인덱스 조회

```sql
SHOW INDEX FROM 테이블명;
```

### 인덱스 삭제

```sql
DROP INDEX idx_이름 ON 직원;
```

---

## 4. 인덱스와 동등 비교 (=)

```sql
-- 인덱스가 있는 컬럼으로 검색하면 빠름
SELECT * FROM 직원 WHERE 이름 = '김민수';
```

- `이름` 컬럼에 인덱스가 있으면 → **인덱스 스캔** (빠름)
- 인덱스가 없으면 → **풀 테이블 스캔** (느림)

---

## 5. 인덱스와 범위 검색

```sql
-- 범위 검색도 인덱스 활용 가능
SELECT * FROM 직원 WHERE 급여 >= 5000 AND 급여 <= 8000;
SELECT * FROM 직원 WHERE 급여 BETWEEN 5000 AND 8000;
```

- 인덱스가 정렬되어 있으니까 **시작점만 찾으면** 나머지는 순서대로 읽음
- `>`, `>=`, `<`, `<=`, `BETWEEN` 모두 인덱스 활용 가능

---

## 6. 인덱스와 LIKE 검색

```sql
-- ✅ 인덱스 사용됨 (앞부분 고정)
SELECT * FROM 직원 WHERE 이름 LIKE '김%';

-- ❌ 인덱스 사용 안 됨 (앞부분이 와일드카드)
SELECT * FROM 직원 WHERE 이름 LIKE '%민수';
SELECT * FROM 직원 WHERE 이름 LIKE '%민%';
```

> ⚠️ **LIKE에서 `%`가 앞에 오면 인덱스를 사용할 수 없음!**
> 이유: 인덱스는 정렬된 상태인데, 앞이 뭔지 모르면 어디서부터 찾아야 할지 모르기 때문

---

## 7. 인덱스와 정렬 (ORDER BY)

```sql
-- 인덱스가 있는 컬럼으로 정렬하면 이미 정렬되어 있어서 빠름
SELECT * FROM 직원 ORDER BY 이름;

-- 인덱스가 없으면 별도 정렬 작업 필요 (느림)
SELECT * FROM 직원 ORDER BY 취미;
```

- 인덱스 자체가 **이미 정렬된 상태**이므로 추가 정렬 불필요
- **filesort** 발생 여부로 확인 가능 (EXPLAIN 사용)

---

## 📌 인덱스 핵심 정리

| 상황 | 인덱스 사용 여부 |
|------|-----------------|
| `WHERE 컬럼 = 값` | ✅ 사용 |
| `WHERE 컬럼 > 값` | ✅ 사용 |
| `WHERE 컬럼 BETWEEN A AND B` | ✅ 사용 |
| `WHERE 컬럼 LIKE '김%'` | ✅ 사용 |
| `WHERE 컬럼 LIKE '%민수'` | ❌ 사용 불가 |
| `WHERE 컬럼 LIKE '%민%'` | ❌ 사용 불가 |
| `ORDER BY 인덱스컬럼` | ✅ 사용 |
| `ORDER BY 비인덱스컬럼` | ❌ 별도 정렬 |
