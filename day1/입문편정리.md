## 1. 데이터베이스 기본 개념

| 용어 | 의미 |
| --- | --- |
| DBMS | 데이터베이스를 관리하는 시스템 (MySQL, PostgreSQL 등) |
| RDBMS | 관계형 DBMS — 테이블(행+열) 구조로 데이터 저장 |
| NoSQL | 비관계형 DB — 유연한 구조 (MongoDB 등) |
| 테이블 | 행(row)과 열(column)로 이루어진 데이터 저장 단위 |
| 행 (Row) | 하나의 데이터 레코드 |
| 열 (Column) | 데이터의 속성 (이름, 나이 등) |
| 기본 키 (PK) | 각 행을 고유하게 식별하는 값 |
| 외래 키 (FK) | 다른 테이블의 기본 키를 참조하는 값 |

---

## 2. SQL 분류

| 분류 | 의미 | 대표 명령어 |
| --- | --- | --- |
| DDL | 데이터 정의 (구조 만들기) | CREATE, ALTER, DROP |
| DML | 데이터 조작 (CRUD) | SELECT, INSERT, UPDATE, DELETE |
| DCL | 데이터 제어 (권한) | GRANT, REVOKE |

---

## 3. DDL — 테이블 관리

### 테이블 생성

```sql
CREATE TABLE 테이블명 (
    컬럼명 데이터타입 제약조건,
    컬럼명 데이터타입 제약조건
);
```

### 주요 데이터 타입

| 타입 | 용도 | 예시 |
| --- | --- | --- |
| INT | 정수 | 나이, 수량 |
| VARCHAR(n) | 가변 문자열 (최대 n자) | 이름, 이메일 |
| TEXT | 긴 문자열 | 본문, 설명 |
| DATE | 날짜 | 2026-03-30 |
| DATETIME | 날짜 + 시간 | 2026-03-30 14:30:00 |
| DECIMAL(m,n) | 소수점 숫자 | 가격, 비율 |
| BOOLEAN | 참/거짓 | TRUE, FALSE |

### 주요 제약 조건

| 제약 조건 | 의미 |
| --- | --- |
| PRIMARY KEY | 기본 키 (고유 + NOT NULL) |
| NOT NULL | 빈 값 허용 안 함 |
| UNIQUE | 중복 허용 안 함 |
| DEFAULT | 기본값 지정 |
| AUTO_INCREMENT | 자동 증가 (보통 PK에 사용) |
| FOREIGN KEY | 외래 키 참조 |

### 테이블 변경

```sql
-- 컬럼 추가
ALTER TABLE 테이블명 ADD 컬럼명 데이터타입;

-- 컬럼 수정
ALTER TABLE 테이블명 MODIFY 컬럼명 새데이터타입;

-- 컬럼 삭제
ALTER TABLE 테이블명 DROP COLUMN 컬럼명;
```

### 테이블 삭제

```sql
DROP TABLE 테이블명;
```

## 4. DML — 데이터 CRUD

### INSERT (등록)

```sql
-- 단건 등록
INSERT INTO 테이블명 (컬럼1, 컬럼2)
VALUES (값1, 값2);

-- 여러 건 등록
INSERT INTO 테이블명 (컬럼1, 컬럼2)
VALUES (값1, 값2), (값3, 값4);
```

### UPDATE (수정)

```sql
UPDATE 테이블명
SET 컬럼1 = 값1, 컬럼2 = 값2
WHERE 조건;
```

> ⚠️ WHERE 없으면 전체 행이 수정됨! 항상 WHERE 쓰기
> 

### DELETE (삭제)

```sql
DELETE FROM 테이블명
WHERE 조건;
```

> ⚠️ WHERE 없으면 전체 행이 삭제됨! 항상 WHERE 쓰기
>

## 5. SELECT — 조회 (가장 많이 쓰는 것!)

### 기본 조회

```sql
-- 전체 컬럼 조회
SELECT * FROM 테이블명;

-- 특정 컬럼만 조회
SELECT 컬럼1, 컬럼2 FROM 테이블명;

-- 별칭 사용
SELECT 컬럼1 AS 별칭 FROM 테이블명;
```

### WHERE — 조건 검색

```sql
-- 기본 비교
WHERE 컬럼 = 값
WHERE 컬럼 != 값  (또는 <>)
WHERE 컬럼 > 값
WHERE 컬럼 >= 값

-- 논리 연산
WHERE 조건1 AND 조건2
WHERE 조건1 OR 조건2
WHERE NOT 조건

-- 편리한 조건들
WHERE 컬럼 BETWEEN 값1 AND 값2     -- 범위
WHERE 컬럼 IN (값1, 값2, 값3)       -- 목록 중 하나
WHERE 컬럼 LIKE '패턴'              -- 패턴 매칭
```

### LIKE 패턴

| 패턴 | 의미 | 예시 |
| --- | --- | --- |
| `%` | 0개 이상의 아무 문자 | `'김%'` → 김으로 시작 |
| `_` | 정확히 1개의 아무 문자 | `'김_'` → 김 + 한 글자 |

## 6. 정렬, 제한, 중복 제거

### ORDER BY — 정렬

```sql
-- 오름차순 (기본)
SELECT * FROM 테이블명 ORDER BY 컬럼 ASC;

-- 내림차순
SELECT * FROM 테이블명 ORDER BY 컬럼 DESC;

-- 여러 기준
SELECT * FROM 테이블명 ORDER BY 컬럼1 DESC, 컬럼2 ASC;
```

### LIMIT — 개수 제한

```sql
-- 상위 5개만
SELECT * FROM 테이블명 LIMIT 5;

-- 3번째부터 5개 (OFFSET)
SELECT * FROM 테이블명 LIMIT 5 OFFSET 3;
```

### DISTINCT — 중복 제거

```sql
SELECT DISTINCT 컬럼 FROM 테이블명;
```

## 7. NULL 처리

```sql
-- NULL 확인 (= 으로 비교 불가!)
WHERE 컬럼 IS NULL
WHERE 컬럼 IS NOT NULL
```

> ⚠️ `WHERE 컬럼 = NULL` 은 안 됨! 반드시 `IS NULL` 사용
>

## 8. 데이터 가공 함수

### 산술 연산

```sql
SELECT 가격 * 수량 AS 총액 FROM 주문;
SELECT 가격 * 1.1 AS 부가세포함 FROM 상품;
```

### 문자열 함수

| 함수 | 용도 | 예시 |
| --- | --- | --- |
| `CONCAT(a, b)` | 문자열 합치기 | `CONCAT('김', '철수')` → 김철수 |
| `SUBSTRING(s, start, len)` | 부분 문자열 | `SUBSTRING('Hello', 1, 3)` → Hel |
| `TRIM(s)` | 앞뒤 공백 제거 | `TRIM(' abc ')` → abc |
| `UPPER(s)` | 대문자로 | `UPPER('hello')` → HELLO |
| `LOWER(s)` | 소문자로 | `LOWER('HELLO')` → hello |
| `LENGTH(s)` | 문자열 길이 | `LENGTH('abc')` → 3 |

### NULL 함수

| 함수 | 용도 |
| --- | --- |
| `IFNULL(컬럼, 대체값)` | NULL이면 대체값 반환 |
| `COALESCE(값1, 값2, ...)` | 첫 번째 NOT NULL 값 반환 |
| `NULLIF(값1, 값2)` | 두 값이 같으면 NULL 반환 |

## 9. 집계 함수

| 함수 | 용도 |
| --- | --- |
| `COUNT(*)` | 행 개수 |
| `COUNT(컬럼)` | NULL 제외한 행 개수 |
| `SUM(컬럼)` | 합계 |
| `AVG(컬럼)` | 평균 |
| `MAX(컬럼)` | 최대값 |
| `MIN(컬럼)` | 최소값 |

## 10. GROUP BY + HAVING

### GROUP BY — 그룹으로 묶기

```sql
SELECT 부서, COUNT(*) AS 인원수
FROM 직원
GROUP BY 부서;
```

### HAVING — 그룹 조건 필터링

```sql
SELECT 부서, AVG(급여) AS 평균급여
FROM 직원
GROUP BY 부서
HAVING AVG(급여) >= 5000;
```

> ⚠️ WHERE vs HAVING 차이:
- `WHERE` → 그룹핑 **전에** 행을 필터링
- `HAVING` → 그룹핑 **후에** 그룹을 필터링
>

## 11. SQL 실행 순서 🔥

> 이거 면접에서 물어봄! 외워두기!
> 

```
FROM       → 어떤 테이블에서?
WHERE      → 어떤 행을 필터링?
GROUP BY   → 어떻게 그룹핑?
HAVING     → 어떤 그룹을 필터링?
SELECT     → 어떤 컬럼을 보여줄까?
ORDER BY   → 어떤 순서로 정렬?
LIMIT      → 몇 개까지 보여줄까?
```

> 작성 순서 (SELECT → FROM → WHERE …) 와 실행 순서가 다르다!
>
